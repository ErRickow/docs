### Cara Menggunakan API

Untuk model penalaran yang menghasilkan respons yang lebih panjang dan terperinci, kami sangat menyarankan untuk melakukan streaming token guna memastikan pengalaman pengguna terbaik.

#### **Perilaku Default (Penalaran Mati)**

Secara default, model akan memberikan jawaban langsung tanpa proses berpikir untuk menghemat token dan mengurangi latensi.

#### **Mengaktifkan Penalaran**

Untuk mengaktifkan penalaran langkah demi langkah, kamu dapat menggunakan **parameter `thinking`** (Direkomendasikan untuk SDK Anthropic/OpenAI) atau menambahkan `/think` ke prompt sistem kamu.

```python icon="python"
from openai import OpenAI

client = OpenAI(
  base_url="https://api.neosantara.xyz/v1",
  api_key="<NAI_API_KEY>"
)

# Mengaktifkan penalaran melalui parameter thinking
stream = client.chat.completions.create(
  model="llama-3.3-nemotron-super-49b-v1.5",
  messages=[{
    "role": "user",
    "content": "Selesaikan teka-teki logika ini: Jika semua mawar adalah bunga dan beberapa bunga berwarna merah, dapatkah kita menyimpulkan bahwa beberapa mawar berwarna merah?",
  }],
  extra_body={
    "thinking": {"type": "enabled", "budget_tokens": 2048}
  },
  temperature=0.7,
  stream=True
)

for chunk in stream:
  if hasattr(chunk.choices[0].delta, 'reasoning_content') and chunk.choices[0].delta.reasoning_content:
      print(f"[Berpikir] {chunk.choices[0].delta.reasoning_content}", end="", flush=True)
  if chunk.choices[0].delta.content:
      print(chunk.choices[0].delta.content, end="", flush=True)
```

#### **Kontrol Manual**

kamu juga dapat mengontrol penalaran melalui instruksi prompt:
- **Aktifkan**: Mulai prompt sistem dengan `/think`.
- **Nonaktifkan (Default)**: Mulai prompt sistem dengan `/no_think`.

### Model Khusus Neosantara

Neosantara menyediakan versi optimal dari model inti seperti `nusantara-base` dan `archipelago-70b` yang secara khusus disetel untuk penalaran. Model-model ini menggunakan mekanisme **Instruction Injection** untuk memberikan pemikiran langkah demi langkah bahkan pada model obrolan standar.

#### **Cara Kerjanya**

Saat kamu mengaktifkan penalaran untuk model-model ini (melalui parameter `thinking` atau `/think`), Neosantara secara otomatis menambahkan instruksi tingkat sistem (misalnya, *"Mohon jelaskan dengan pemikiran langkah demi langkah"*) ke permintaan kamu. Ini memastikan model membedah masalah sebelum memberi kamu jawaban akhir.

### Model Berpikir (Claude & Gemini)

Untuk model yang mendukung penalaran asli, seperti **Claude 3.5 Sonnet** and **Gemini 2.0 Flash**, Neosantara mendukung parameter `thinking`. Ini memungkinkan kamu untuk mengalokasikan anggaran token tertentu untuk proses pemikiran internal model.

#### **Menggunakan Parameter `thinking`**

Saat menggunakan API yang kompatibel dengan Anthropic, kamu dapat memberikan objek `thinking`.

```json icon="code"
{
  "model": "claude-3-haiku",
  "messages": [...],
  "thinking": {
    "type": "enabled",
    "budget_tokens": 2048
  }
}
```

#### **Pemetaan Upaya Penalaran**

Jika kamu menggunakan API yang kompatibel dengan OpenAI, anggaran `thinking` secara otomatis dipetakan ke parameter `reasoning_effort` berdasarkan standar industri:

| `budget_tokens` | `reasoning_effort` |
| :--- | :--- |
| >= 10,000 | `high` |
| >= 5,000 | `medium` |
| >= 2,000 | `low` |
| `< 2,000` | `minimal` |

### Praktik Terbaik

Untuk mendapatkan hasil terbaik dari Llama 3.3 Nemotron, perlakukan ia seperti pemecah masalah ahli. Berikan tujuan tingkat tinggi yang jelas dan biarkan model menentukan langkah terbaik untuk mencapai solusi.

- **Kekuatan**: Unggul dalam penalaran terbuka, logika multi-langkah, dan masalah pengodean atau matematika yang kompleks.
- **Hindari Over-prompting**: Mengatur setiap langkah secara mikro dapat membatasi kemampuan penalaran tingkat lanjut model. Berikan tujuannya, bukan jalan pastinya.
- **Berikan Tujuan yang Jelas**: Pastikan prompt kamu jelas dan tidak ambigu untuk mendapatkan respons yang paling akurat dan relevan.
- **Gunakan Streaming**: Untuk kueri yang kompleks, proses penalaran dapat menghasilkan banyak teks. Melakukan streaming respons memberikan pengalaman pengguna yang jauh lebih baik.
