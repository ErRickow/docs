Model penalaran (reasoning) dirancang untuk "berpikir" sebelum berbicara. Berbeda dengan model chat standar yang memprediksi token berikutnya secara instan, model penalaran menghasilkan urutan pemikiran internal (chain-of-thought) untuk membedah instruksi kompleks, memverifikasi fakta, dan mengoreksi logika mereka sendiri sebelum menyajikan jawaban akhir.

## Field `reasoning_content`

Neosantara menyediakan antarmuka terpadu untuk penalaran di seluruh penyedia model. Terlepas dari apakah model menggunakan blok native atau tag khusus (seperti `<think>`), Neosantara menormalkan output ini ke dalam field khusus **`reasoning_content`** dalam respons API.

---

## Cara Kerjanya

Neosantara menggunakan tiga mekanisme berbeda untuk mengaktifkan penalaran, tergantung pada kemampuan model tersebut.

### 1. Penalaran Native (Asli)
Model yang dilatih khusus untuk penalaran, seperti **DeepSeek-R1**, **Claude 3.7**, dan **Gemini 2.0 Flash Thinking**, menggunakan logika bawaan mereka. Neosantara terhubung langsung ke protokol "berpikir" native ini untuk memastikan kualitas pemikiran tertinggi.

### 2. Injeksi Instruksi (Instruction Injection)
Untuk model standar yang tidak memiliki penalaran native (misalnya, `nusantara-base`, `archipelago-70b`), Neosantara menggunakan **Injeksi Instruksi**. Saat penalaran diaktifkan, sistem secara otomatis menambahkan instruksi logika kompleks ke prompt sistem, memaksa model untuk membedah prosesnya ke dalam beberapa langkah.

### 3. Anggaran & Upaya Penalaran
Untuk model yang didukung, kamu dapat mengontrol "kedalaman" pemikiran menggunakan parameter `thinking`.

| Anggaran (Token) | Tingkat Upaya | Kasus Penggunaan |
| :--- | :--- | :--- |
| < 2.000 | `minimal` | Pemeriksaan logika cepat, ringkasan sederhana. |
| 2.000 - 5.000 | `low` | Tugas pengodean dasar, instruksi multi-langkah. |
| 5.000 - 10.000 | `medium` | Debugging kompleks, logika penulisan kreatif. |
| > 10.000 | `high` | Pembuktian matematis, perencanaan arsitektur. |

## Mengaktifkan Penalaran

Kamu dapat mengaktifkan penalaran menggunakan objek konfigurasi `thinking` atau pemicu prompt manual.

### Melalui Parameter API (Direkomendasikan)

<CodeGroup>

```python OpenAI SDK
response = client.chat.completions.create(
    model="deepseek-chat-v3.1",
    messages=[{"role": "user", "content": "Selesaikan: 15 * 24 + 133"}],
    extra_body={
        "thinking": { "type": "enabled", "budget_tokens": 2048 }
    }
)

# Akses penalaran secara terpisah
print(response.choices[0].message.reasoning_content)
print(response.choices[0].message.content)
```

```json JSON Payload
{
  "model": "claude-4.5-sonnet",
  "messages": [...],
  "thinking": {
    "type": "enabled",
    "budget_tokens": 4096
  }
}
```

</CodeGroup>

### Melalui Pemicu Manual
Kamu juga dapat memaksa model chat apa pun ke mode penalaran dengan memulai prompt sistem kamu dengan perintah **`/think`**. Untuk menonaktifkannya secara eksplisit, gunakan **`/no_think`**.

## Praktik Terbaik

1. **Gunakan Streaming**: Proses penalaran dapat menghasilkan ribuan token internal sebelum jawaban akhir muncul. Gunakan [Streaming](/id/capability/stream) untuk memberikan umpan balik visual waktu nyata ("Sedang berpikir...") kepada pengguna kamu.
2. **Spesifik, Bukan Deskriptif**: Berikan tujuan yang jelas kepada model penalaran, tetapi hindari mengatur setiap langkahnya secara mikro. Biarkan model menentukan jalur optimal menuju solusi.
3. **Atur Anggaran dengan Bijak**: Lebih banyak token penalaran akan meningkatkan latensi dan biaya. Hanya alokasikan anggaran tinggi (>10rb) untuk tugas sains atau matematika yang benar-benar kompleks.
