Agno adalah framework sumber terbuka untuk membangun agent, team, dan aplikasi AI multimodal. Neosantara tersedia sebagai provider model native Agno melalui `agno.models.neosantara.Neosantara`, sehingga Kamu dapat memakai model Neosantara langsung di dalam Agno tanpa membuat model OpenAI-compatible custom.

Gunakan integrasi ini saat Kamu ingin memakai runtime agent Agno, tools, streaming response, memory, atau workflow team sambil merutekan panggilan model melalui Neosantara.

## Instal

Instal Agno dan tool yang ingin Kamu pakai. Contoh di bawah menyertakan DuckDuckGo search.

```bash icon="terminal"
pip install -U agno ddgs
```

## Autentikasi

Setel API key Neosantara Kamu.

```bash icon="terminal"
export NEOSANTARA_API_KEY="nsk_..."
```

Provider native memakai `https://api.neosantara.xyz/v1` secara default. Kamu juga dapat meneruskan `api_key` atau `base_url` langsung ke `Neosantara(...)` saat membutuhkan konfigurasi runtime eksplisit.

## Agent Dasar

Buat `Agent` Agno dan kirim instance model native Neosantara.

```python icon="python"
from agno.agent import Agent
from agno.models.neosantara import Neosantara

agent = Agent(
    model=Neosantara(id="claude-opus-4-6"),
    markdown=True,
)

agent.print_response("Jelaskan fungsi AI gateway dalam dua kalimat.")
```

## Streaming Response

Gunakan `stream=True` untuk assistant terminal, dashboard, dan UI chat yang membutuhkan output bertahap.

```python icon="python"
from agno.agent import Agent
from agno.models.neosantara import Neosantara

agent = Agent(
    model=Neosantara(id="claude-opus-4-6"),
    markdown=True,
)

agent.print_response(
    "Buat checklist respons insiden singkat untuk outage API.",
    stream=True,
)
```

## Agent dengan Tools

Tools Agno bekerja dengan class model native Neosantara. Gunakan model yang mendukung tool calling, lalu pasang tools ke agent.

```python icon="python"
from agno.agent import Agent
from agno.models.neosantara import Neosantara
from agno.tools.duckduckgo import DuckDuckGoTools

agent = Agent(
    model=Neosantara(id="claude-opus-4-6"),
    tools=[DuckDuckGoTools()],
    instructions=[
        "Cari di web hanya saat jawaban membutuhkan informasi terbaru.",
        "Sebutkan nama sumber paling relevan di jawaban akhir.",
    ],
    markdown=True,
)

agent.print_response("Apa perubahan terbaru di ekosistem AI Indonesia?", stream=True)
```

## Konfigurasi Parameter Model

Provider native menerima model id Neosantara dan parameter model OpenAI-compatible umum yang didukung Agno.

```python icon="python"
from agno.agent import Agent
from agno.models.neosantara import Neosantara

model = Neosantara(
    id="claude-opus-4-6",
    temperature=0.2,
    max_tokens=500,
)

agent = Agent(
    model=model,
    markdown=True,
)

agent.print_response("Buat draft balasan support untuk pembayaran gagal.")
```

Untuk deployment privat atau gateway testing, teruskan base URL eksplisit.

```python icon="python"
from agno.models.neosantara import Neosantara

model = Neosantara(
    id="claude-opus-4-6",
    api_key="nsk_...",
    base_url="https://api.neosantara.xyz/v1",
)
```

## Pemilihan Model

Pilih model berdasarkan tugas agent:

| Tugas agent | Tipe model yang disarankan |
| --- | --- |
| Chat umum dan support | Model teks seperti `claude-opus-4-6` |
| Workflow berat tools | Model tool-calling dengan dukungan function call stabil |
| Reasoning dan planning | Model reasoning dari [Models Overview](/id/models-overview) |
| Analisis vision atau multimodal | Halaman model dengan dukungan input image |

Buka [Models Overview](/id/models-overview) untuk mengonfirmasi context window, endpoint, unit harga, dan capability flags sebelum menampilkan model di dokumentasi atau contoh production.

## Kapan Menggunakan Agno

| Use case | Alasan Agno membantu |
| --- | --- |
| Aplikasi agent | Runtime `Agent`, instructions, tools, dan streaming sudah tersedia. |
| Workflow tools | Pasang search, database, API, atau custom tools di sekitar model native. |
| Prototype multimodal | Pakai struktur agent Agno yang model-agnostic sambil memilih model Neosantara yang sesuai. |
| Sistem multi-agent | Bergerak dari satu agent ke team dan workflow tanpa mengganti provider model. |

## Troubleshooting

| Gejala | Cek |
| --- | --- |
| API key tidak ditemukan | Setel `NEOSANTARA_API_KEY` atau teruskan `api_key` ke `Neosantara(...)`. |
| Tool calls tidak berjalan | Pastikan model mendukung tool calling dan paket tool sudah terinstal. |
| Agent tidak bisa mengakses informasi terbaru | Tambahkan search tool seperti `DuckDuckGoTools()` dan berikan instruksi pencarian yang jelas. |
| Perilaku model tidak sesuai ekspektasi | Cek halaman model generated untuk capability, context limit, dan known limitations. |
