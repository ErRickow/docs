## Pendahuluan

API File memungkinkan Kamu mengunggah dan mengelola file yang dapat digunakan dengan berbagai fitur seperti pemrosesan batch, asisten, dan fine-tuning model. File disimpan dengan aman dan dapat diakses melalui API REST yang sederhana.

### Fitur Utama

- **Beberapa Tujuan**: Dukungan untuk alur kerja pemrosesan batch, asisten, dan fine-tuning.
- **Validasi JSONL**: Validasi otomatis untuk file pemrosesan batch.
- **Penyimpanan Aman**: File disimpan dengan enkripsi dan kontrol akses.
- **Siklus Hidup File**: Masa berlaku otomatis dan pembersihan file lama.
- **Batas Ukuran**: Mendukung file hingga 100MB.

## Mulai Cepat

### 1. Unggah File

```bash
curl https://api.neosantara.xyz/v1/files \
  -H "Authorization: Bearer KUNCI_API_ANDA" \
  -F purpose="batch" \
  -F file="@batch_requests.jsonl"
```

### 2. Daftar File Kamu

```bash
curl https://api.neosantara.xyz/v1/files \
  -H "Authorization: Bearer KUNCI_API_ANDA"
```

### 3. Unduh Konten File

```bash
curl https://api.neosantara.xyz/v1/files/file-abc123/content \
  -H "Authorization: Bearer KUNCI_API_ANDA" \
  --output downloaded_file.jsonl
```

## Persyaratan File

### Persyaratan Umum

- Ukuran file maksimum: **100MB**
- File disimpan selama **24 jam** secara default.
- Setiap file harus memiliki tujuan (*purpose*) yang ditentukan.

### Format JSONL untuk Pemrosesan Batch

Saat mengunggah file dengan `purpose="batch"`, pastikan file JSONL Kamu mengikuti format berikut:

```json example.jsonl
{"custom_id": "request-1", "method": "POST", "url": "/v1/chat/completions", "body": {"model": "garda-beta-mini", "messages": [{"role": "user", "content": "Halo!"}]}}
{"custom_id": "request-2", "method": "POST", "url": "/v1/chat/completions", "body": {"model": "garda-beta-mini", "messages": [{"role": "user", "content": "Apa kabar?"}]}}
```

Setiap baris harus berupa objek JSON yang valid dengan:

- `custom_id`: Pengenal unik untuk permintaan.
- `method`: Metode HTTP (biasanya "POST").
- `url`: Jalur endpoint API.
- `body`: Payload permintaan.

<Note>
File yang diunggah untuk pemrosesan batch harus dalam format JSONL yang valid. Setiap baris harus berupa objek JSON yang lengkap.
</Note>

## Penanganan Kesalahan

Kesalahan umum yang mungkin Kamu temui:

| Kode Kesalahan | Deskripsi | Solusi |
| ------------ | ------------- | ---------- |
| `missing_file` | Parameter file tidak diberikan | Pastikan Kamu mengirimkan file dalam permintaan |
| `invalid_purpose` | Nilai tujuan tidak valid | Gunakan salah satu dari: `batch`, `assistants`, atau `fine-tune` |
| `file_too_large` | File melebihi batas 100MB | Bagi file besar atau kompres kontennya |
| `invalid_file_format` | Validasi JSONL gagal | Periksa format JSONL Kamu untuk kesalahan sintaksis |
| `file_not_found` | File tidak ditemukan | Verifikasi bahwa file_id sudah benar |
| `file_expired` | File telah kedaluwarsa | Unggah ulang file jika diperlukan |
