Batas penggunaan kami berbasis token, memastikan pendekatan yang adil dan transparan untuk penggunaan API. Sistem ini menjaga kualitas layanan untuk semua pengguna dengan mencegah penggunaan sumber daya berlebihan. Batas diterapkan pada **token per hari** dan **token per bulan** (disimpan di database kami). Batas tambahan untuk permintaan per hari dan per menit diterapkan secara terprogram menggunakan Upstash Redis. Lihat [Kredit Token & Harga](/token-credits) untuk detail lebih lanjut.

## Header Respons

Setiap respons API menyertakan informasi penggunaan dalam kolom `_metadata`. Ini memungkinkan Anda melacak konsumsi saat ini terhadap batas paket langsung dari aplikasi Anda.

```json
{
  "_metadata": {
    "usage": {
      "daily": {
        "current": 50,
        "limit": 1500, // Contoh batas untuk tingkatan Gratis
        "reset_date": "2025-07-28T00:00:00Z",
        "remaining": 1450
      },
      "monthly": {
        "current": 500,
        "limit": 7500, // Contoh batas untuk tingkatan Gratis
        "reset_date": "2025-08-01T00:00:00Z",
        "remaining": 7000
      }
    },
    "tier": {
      "name": "Free",
      "priority": 1
    }
  }
}
```

## Kesalahan Batas Penggunaan

Jika penggunaan API Anda melebihi batas yang dialokasikan untuk paket Anda, API akan mengembalikan Kode Status HTTP `429: Terlalu Banyak Permintaan`. Badan respons akan mencakup detail tentang batas yang terlampaui dan kapan akan diatur ulang.

```json
{
  "status": false,
  "creator": "neosantara.xyz",
  "error": {
    "message": "Daily token limit exceeded for your account (1500).",
    "type": "usage_limit_exceeded",
    "code": "user_total_daily_limit_exceeded"
  },
  "usage": {
    "current": 1500,
    "limit": 1500,
    "reset_date": "2025-07-28T00:00:00Z",
    "remaining": 0
  },
  "tier": "Free",
  "timestamp": "2025-07-27T04:10:59Z"
}
```
