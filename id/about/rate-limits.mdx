Neosantara AI menerapkan batas throughput untuk memastikan stabilitas platform dan alokasi sumber daya yang adil. Sistem ini mengelola laju permintaan masuk dan pemrosesan token.

Untuk informasi mengenai penagihan, saldo kredit, dan migrasi kuota, silakan merujuk ke dokumentasi [Kredit Token & Harga](/id/about/token-credits).

## Batas Throughput (Kecepatan)

Batas throughput membatasi laju operasi dalam jendela waktu tertentu. Ini adalah batasan teknis yang dirancang untuk mencegah kelebihan beban sistem dan berbeda dari batas penagihan.

Sistem memberlakukan tiga jenis batas throughput:
*   **RPM (Requests Per Minute):** Jumlah maksimum permintaan HTTP yang diizinkan per menit.
*   **ITPM (Input Tokens Per Minute):** Jumlah maksimum token input yang diproses per menit.
*   **OTPM (Output Tokens Per Minute):** Jumlah maksimum token output yang dihasilkan per menit.

### Batas API Standar

| Tingkatan | RPM | ITPM | OTPM |
| :--- | :-- | :-- | :-- |
| Free | 3 | 5.000 | 2.000 |
| Basic | 50 | 20.000 | 5.000 |
| Standard | 1.000 | 100.000 | 25.000 |
| Pro | 2.000 | 200.000 | 50.000 |
| Enterprise | 4.000 | 500.000 | 125.000 |

### Batas CLI

| Tingkatan | RPM | TPM (Total Token/menit) |
| :--- | :-- | :-- |
| Free | 10 | 10.000 |
| Basic | 60 | 50.000 |
| Standard | 1.000 | 100.000 |
| Pro | 2.000 | 200.000 |
| Enterprise | Unlimited | Unlimited |

<Note>
**Peningkatan Tingkatan:** Tingkatan throughput disesuaikan secara otomatis berdasarkan riwayat total deposit Anda. Lihat [Kredit Token & Harga](/id/about/token-credits#tingkatan-throughput-batas-kecepatan) untuk detailnya.
</Note>

## Header Respons

Respons API menyertakan header HTTP standar yang memberikan status waktu nyata penggunaan batas rate client. Client harus memantau header ini untuk menghindari pelampauan batas.

| Header | Deskripsi |
| :--- | :--- |
| `x-neosantara-ratelimit-requests-limit` | RPM maksimum yang diizinkan. |
| `x-neosantara-ratelimit-requests-remaining` | Jumlah permintaan yang tersisa dalam jendela saat ini. |
| `x-neosantara-ratelimit-requests-reset` | Waktu (ISO 8601) ketika batas permintaan direset. |
| `x-neosantara-ratelimit-input-tokens-limit` | ITPM maksimum yang diizinkan. |
| `x-neosantara-ratelimit-input-tokens-remaining` | Jumlah token input yang tersisa dalam jendela saat ini. |
| `x-neosantara-ratelimit-input-tokens-reset` | Waktu (ISO 8601) ketika batas token input direset. |
| `x-neosantara-ratelimit-output-tokens-limit` | OTPM maksimum yang diizinkan. |
| `x-neosantara-ratelimit-output-tokens-remaining` | Jumlah token output yang tersisa dalam jendela saat ini. |
| `x-neosantara-ratelimit-output-tokens-reset` | Waktu (ISO 8601) ketika batas token output direset. |
| `x-neosantara-tier` | Tingkatan throughput yang ditetapkan untuk akun saat ini. |

## Penanganan Kesalahan

### 429 Too Many Requests
Kesalahan ini menunjukkan bahwa salah satu **Batas Throughput** teknis (RPM, ITPM, atau OTPM) telah terlampaui. Respons menyertakan header `Retry-After` yang menentukan waktu tunggu dalam detik sebelum permintaan baru dapat dibuat.

```json
{
  "error": {
    "message": "Too many requests. Please try again later.",
    "type": "rate_limit_exceeded",
    "code": "rpm_exceeded",
    "details": {
        "retry_after": 5,
        "limit": 50,
        "remaining": 0,
        "reset": "2025-12-16T10:30:05.000Z"
    }
  }
}
```
