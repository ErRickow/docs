Untuk informasi mengenai penagihan dan kredit, silakan lihat [Kredit Token & Harga](/id/about/token-credits).

## Batas Throughput (Kecepatan)

Sistem menerapkan tiga jenis batasan:

* **RPM (Requests Per Minute):** Maksimal jumlah permintaan HTTP per menit.
* **ITPM (Input Tokens Per Minute):** Maksimal jumlah token input per menit.
* **OTPM (Output Tokens Per Minute):** Maksimal jumlah token output per menit.

### Batas API Standar

| Tier | RPM | ITPM | OTPM |
| :--- | :--- | :--- | :--- |
| Free | 3 | 5.000 | 2.000 |
| Basic | 50 | 20.000 | 5.000 |
| Standard | 1.000 | 100.000 | 25.000 |
| Pro | 2.000 | 200.000 | 50.000 |
| Enterprise | 4.000 | 500.000 | 125.000 |

<Note>
**Upgrade Tier:** Tier akun disesuaikan otomatis berdasarkan riwayat total deposit Kamu.
</Note>

## Response Headers

Setiap respon API menyertakan header HTTP untuk memantau sisa kuota penggunaan Kamu secara waktu nyata.

| Header | Deskripsi |
| :--- | :--- |
| `x-neosantara-ratelimit-requests-limit` | Batas RPM maksimal. |
| `x-neosantara-ratelimit-requests-remaining` | Sisa jumlah permintaan dalam jendela waktu saat ini. |
| `x-neosantara-ratelimit-requests-reset` | Waktu (ISO 8601) saat batas permintaan direset. |
| `x-neosantara-tier` | Tier throughput yang saat ini ditetapkan untuk akun tersebut. |

## Penanganan Error

### 429 Too Many Requests

Error ini menunjukkan bahwa salah satu batas teknis (RPM, ITPM, atau OTPM) telah terlampaui. Respon akan menyertakan header `Retry-After` yang menentukan waktu tunggu dalam detik sebelum permintaan baru dapat dilakukan.

```json icon="code"
{
  "error": {
    "message": "Too many requests. Please try again later.",
    "type": "rate_limit_exceeded",
    "code": "rpm_exceeded",
    "details": {
        "retry_after": 5,
        "limit": 50,
        "remaining": 0,
        "reset": "2025-12-16T10:30:05.000Z"
    }
  }
}
```
