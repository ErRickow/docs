Model Context Protocol, or MCP, is an open standard that enables AI assistants to securely access local or remote tools and data sources. Neosantara provides a native, out-of-the-box MCP server running directly on the API gateway.

Connecting your local coding environment to the Neosantara MCP server allows tools like Cursor or Claude Desktop to call Neosantara models, generate images, discover available models, and access rich agentic capabilities directly from your IDE or chat client.

---

## 1. Create an MCP API Key

To access the MCP server, you need a dedicated MCP API key. Standard Neosantara keys cannot be used to authenticate MCP endpoints.

<Steps>
  <Step title="Navigate to the Dashboard">
    Go to the API Keys management page on the Neosantara Dashboard at [app.neosantara.xyz/api-keys](https://app.neosantara.xyz/api-keys).
  </Step>
  <Step title="Create the Key">
    Click on the **Create Key** or **Create MCP Key** button.
  </Step>
  <Step title="Save the Key">
    Save the generated key securely. Your key will have the prefix `nsk_mcp_`.
  </Step>
</Steps>

<Warning>
  MCP API keys are shown only once upon creation. Keep it secure and do not share it.
</Warning>

## 2. Supported Tools

Once connected, the Neosantara MCP server exposes the following tools to your AI assistant:

| Tool Name | Description |
| --- | --- |
| `chat` | Generate text completions and model responses using Neosantara's Responses API. This supports rich payloads, tools, and system instructions. |
| `list-models` | Discovers active language models available on Neosantara with details on capabilities, context windows, and pricing. |
| `generate-image` | Generates high-quality images from text descriptions. |
| `list-image-models` | Lists available image generation models including Google Imagen 4.0 and Stability AI SDXL. |

## 3. Configuration & Integration

Neosantara’s MCP server uses the Server-Sent Events transport protocol. This is also referred to as SSE. It allows you to connect directly via a URL without downloading any local executable scripts.

* **Endpoint URL**: `https://api.neosantara.xyz/v1/mcp`
* **Authorization Header**: `x-api-key`

### Cursor IDE

To connect Neosantara tools to Cursor:

<Steps>
  <Step title="Open Settings">
    Open Cursor Settings. You can access this via the keyboard shortcut `Ctrl + ,` on Windows or `Cmd + ,` on macOS.
  </Step>
  <Step title="Navigate to MCP">
    Go to **Features** and select **MCP**.
  </Step>
  <Step title="Add Server">
    Click on the **+ Add New MCP Server** button.
  </Step>
  <Step title="Configure Server Settings">
    Configure the fields as follows:
    * **Name**: `neosantara`
    * **Type**: `SSE`
    * **URL**: `https://api.neosantara.xyz/v1/mcp`
  </Step>
  <Step title="Add Authentication Header">
    Click **+ Add Header** to add the authorization header:
    * **Header Name**: `x-api-key`
    * **Header Value**: `YOUR_NSK_MCP_KEY`
  </Step>
  <Step title="Save and Verify">
    Click **Save**. The server status should turn green and show 4 tools active.
  </Step>
</Steps>

### Claude Desktop

To connect Neosantara tools to the Claude Desktop application:

<Steps>
  <Step title="Open Configuration File">
    Open your Claude Desktop configuration file at one of the following paths:
    * **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
    * **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
  </Step>
  <Step title="Add Neosantara MCP Server">
    Add `neosantara` to the `mcpServers` list using the SSE configuration:

    ```json
    {
      "mcpServers": {
        "neosantara": {
          "url": "https://api.neosantara.xyz/v1/mcp",
          "headers": {
            "x-api-key": "YOUR_NSK_MCP_KEY"
          }
        }
      }
    }
    ```
  </Step>
  <Step title="Restart Claude Desktop">
    Restart the Claude Desktop application. You will see a new hammer icon in the chat input representing active tools.
  </Step>
</Steps>

### CLI Tools and Terminal Agents

You can also integrate the Neosantara MCP server with CLI-based coding assistants and terminal agents.

#### Gemini CLI

To add Neosantara as an MCP server in the Gemini CLI, run the following command:

```bash
gemini mcp add neosantara https://api.neosantara.xyz/v1/mcp -t http -H "x-api-key: YOUR_NSK_MCP_KEY"
```

To verify the connection, you can list the configured MCP servers:

```bash
gemini mcp list
```

#### Claude Code

To add Neosantara as an MCP server in Claude Code, run the add command with the transport HTTP flag:

```bash
claude mcp add --transport http neosantara https://api.neosantara.xyz/v1/mcp --header "x-api-key: YOUR_NSK_MCP_KEY"
```

To view the connection status, list the active MCP servers:

```bash
claude mcp list
```

#### Codex CLI

To add Neosantara as an MCP server in Codex, use the add command and specify your key:

```bash
codex mcp add neosantara --url https://api.neosantara.xyz/v1/mcp --bearer-token-env-var NAI_MCP_KEY
```

Make sure to export the environment variable `NAI_MCP_KEY` containing your Neosantara MCP key in your shell configuration.

#### Kiro CLI

To connect Neosantara to Kiro CLI, open your configuration file:

* **Path**: `~/.kiro/settings/mcp.json`

Add the Neosantara configuration under the `mcpServers` object:

```json
{
  "mcpServers": {
    "neosantara": {
      "url": "https://api.neosantara.xyz/v1/mcp",
      "headers": {
        "x-api-key": "YOUR_NSK_MCP_KEY"
      }
    }
  }
}
```

## 5. Rate Limits

MCP endpoints have separate throughput limits and daily request quotas that are enforced based on your account's subscription tier. For example, Free keys get 20 requests per minute and 500 daily requests.

For the complete tier-by-tier quota breakdown and details, please refer to the main [Rate Limits](/en/about/rate-limits#mcp-api-limits) documentation page.
