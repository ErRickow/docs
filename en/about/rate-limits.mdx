Neosantara implements throughput limits to ensure platform stability and fair resource allocation. This system manages the rate of incoming requests and token processing.

For information on billing and credit balances, please refer to the [Token Credits & Pricing](/en/about/billing-pricing) documentation.

---

## Throughput Limits (Speed)

Throughput limits restrict the rate of operations over a specific time window. These are technical constraints designed to prevent system overload and are distinct from billing limits.

The system enforces three types of throughput limits:

* **RPM (Requests Per Minute):** The maximum number of HTTP requests allowed per minute.
* **ITPM (Input Tokens Per Minute):** The maximum number of input tokens processed per minute.
* **OTPM (Output Tokens Per Minute):** The maximum number of output tokens generated per minute.

### Standard API Limits

| Tier | RPM | ITPM | OTPM |
| :--- | :--- | :--- | :--- |
| Free | 15 | 30,000 | 8,000 |
| Basic | 50 | 500,000 | 80,000 |
| Standard | 1,000 | 2,000,000 | 320,000 |
| Pro | 2,000 | 5,000,000 | 800,000 |
| Enterprise | 4,000 | 10,000,000 | 1,600,000 |

<Note>
**Tier Upgrades:** Throughput tiers are automatically adjusted based on your total deposit history. See [Token Credits & Pricing](/en/about/billing-pricing#throughput-tiers-speed-limits) for details.
</Note>

### MCP API Limits

For endpoints under `/v1/mcp`, separate throughput and daily limits apply based on the account's throughput tier:

| Tier | RPM | Daily Request Limit |
| :--- | :--- | :--- |
| Free | 20 | 500 |
| Basic | 40 | 1,500 |
| Standard | 60 | 3,000 |
| Pro | 120 | 8,000 |
| Enterprise | 240 | 20,000 |

<Note>
  MCP limits are enforced per API key. Additionally, user-level aggregate limits across all of your active MCP keys are applied to ensure platform stability.
</Note>

## Response Headers

API responses include standard HTTP headers that provide real-time status of the client's rate limit usage. Clients should monitor these headers to avoid hitting rate limits.

| Header | Description |
| :--- | :--- |
| `x-neosantara-ratelimit-requests-limit` | The maximum RPM allowed. |
| `x-neosantara-ratelimit-requests-remaining` | The number of requests remaining in the current window. |
| `x-neosantara-ratelimit-requests-reset` | The time (ISO 8601) when the request limit resets. |
| `x-neosantara-ratelimit-input-tokens-limit` | The maximum ITPM allowed. |
| `x-neosantara-ratelimit-input-tokens-remaining` | The number of input tokens remaining in the current window. |
| `x-neosantara-ratelimit-input-tokens-reset` | The time (ISO 8601) when the input token limit resets. |
| `x-neosantara-ratelimit-output-tokens-limit` | The maximum OTPM allowed. |
| `x-neosantara-ratelimit-output-tokens-remaining` | The number of output tokens remaining in the current window. |
| `x-neosantara-ratelimit-output-tokens-reset` | The time (ISO 8601) when the output token limit resets. |
| `x-neosantara-tier` | The current throughput tier assigned to the account. |

## Error Handling

### 429 Too Many Requests

This error indicates that one of the technical **Throughput Limits** (RPM, ITPM, or OTPM) has been exceeded. The response includes a `Retry-After` header specifying the wait time in seconds before a new request can be made.

```json icon="code"
{
  "error": {
    "message": "Too many requests. Please try again later.",
    "type": "rate_limit_exceeded",
    "code": "rpm_exceeded",
    "details": {
        "retry_after": 5,
        "limit": 50,
        "remaining": 0,
        "reset": "2025-12-16T10:30:05.000Z"
    }
  }
}
```
