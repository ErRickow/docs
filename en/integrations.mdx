Use this page to choose an integration path. Neosantara works through native framework providers where available, OpenAI-compatible adapters for broad ecosystem support, and Anthropic-compatible APIs for Claude-style workflows.

## Recommended paths

<CardGroup cols={2}>
  <Card title="Agno native" icon="sparkles" href="/en/agno">
    Use `agno.models.neosantara.Neosantara` for multimodal agents, tool use, streaming, and model switching.
  </Card>

  <Card title="Any-LLM native" icon="blocks" href="/en/any-llm">
    Use provider id `neosantara` with `NEOSANTARA_API_KEY` when you want one Python interface across providers.
  </Card>

  <Card title="LiteLLM native" icon="bot" href="/en/litellm">
    Use the `neosantara/` model prefix for LiteLLM routing, Responses API calls, and mixed-provider workflows.
  </Card>
</CardGroup>

## OpenAI-compatible adapters

These frameworks can use Neosantara by pointing their OpenAI-compatible client to `https://api.neosantara.xyz/v1`.

<CardGroup cols={3}>
  <Card title="LangChain" icon="link" href="/en/langchain">
    Use `langchain-openai` with the Neosantara base URL.
  </Card>

  <Card title="CrewAI" icon="users" href="/en/crewai">
    Run CrewAI agents through an OpenAI-compatible Neosantara model string.
  </Card>

  <Card title="DSPy" icon="route" href="/en/dspy">
    Configure DSPy with `openai/<model-id>` and the Neosantara API base.
  </Card>

  <Card title="Pydantic AI" icon="badge-check" href="/en/pydantic-ai">
    Use Pydantic AI with Neosantara for typed agent and structured output workflows.
  </Card>

  <Card title="LlamaIndex" icon="database" href="/en/llama-index">
    Connect LlamaIndex RAG workflows to Neosantara chat and embedding models.
  </Card>

  <Card title="AutoGen / AG2" icon="bot" href="/en/autogen">

  </Card>
</CardGroup>
