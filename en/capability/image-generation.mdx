Neosantara API provides a unified endpoint for image generation, fully compatible with the OpenAI standard.

## Supported Models

<ModelCards capability="image_generation" />

## Usage

Send a `POST` request to `/v1/images/generations`.

<Note>
  See the [API Reference](/api-reference/images/create-image-generation) for full parameter details.
</Note>

### Image-to-Image & Inpainting

Some models support advanced tasks:
*   **Variation**: Provide a base64 image in the `image` parameter.
*   **Inpainting**: Modify specific areas using a `mask_image`.
*   **Outpainting**: Extend images using `outpainting_mode`.

---

## Technical Architecture

Neosantara's image pipeline is built for reliability and security, supporting two primary processing modes.

### Synchronous vs Asynchronous
*   **Standard API (`/v1/images/generations`)**: Best for real-time applications. The request stays open until the image is generated.
*   **Responses API (`/v1/responses`)**: Recommended for slow models or high-resolution batches. Set `background: true` to receive a `response_id` and poll for results, preventing HTTP timeouts.

### Delivery & Caching
All generated images are delivered via an **Ephemeral CDN**.
1.  Images are temporarily stored in a high-speed Redis cache.
2.  The API returns a signed, temporary URL (e.g., `https://api.neosantara.xyz/share/generated/...`).
3.  **TTL (Time-to-Live)**: Links are valid for **5 minutes**. Users should download or re-host images if permanent storage is required.

### Security (SSRF Protection)
When providing an `image_url` for variations or inpainting, Neosantara's **Security Bridge** validates the URL:
*   Resolves DNS to verify IP ranges.
*   Blocks all private, loopback, and reserved network addresses to prevent Server-Side Request Forgery (SSRF).

### Tier-Based Logic
*   **Paid Tiers**: Receive original, high-fidelity images.
*   **Free Tier**: Images are automatically processed through a **Watermark Engine** before delivery to comply with usage policies.

## Examples

<CodeGroup>

```python OpenAI (Python)
from openai import OpenAI
import os

client = OpenAI(
    api_key=os.environ.get("YOUR_NEOSANTARA_API_KEY"),
    base_url="https://api.neosantara.xyz/v1"
)

response = client.images.generate(
    model="imagen-4.0-fast",
    prompt="A futuristic laboratory, high-tech, extremely detailed.",
    n=1,
    size="1024x1024",
    response_format="url"
)

print(response.data[0].url)
```

```javascript OpenAI (Node.js)
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: "YOUR_NEOSANTARA_API_KEY",
  baseURL: "https://api.neosantara.xyz/v1",
});

const response = await client.images.generate({
  model: "imagen-4.0-fast",
  prompt: "A futuristic laboratory, high-tech, extremely detailed.",
  n: 1,
  size: "1024x1024",
});

console.log(response.data[0].url);
```
